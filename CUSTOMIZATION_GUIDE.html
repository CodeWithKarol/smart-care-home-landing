1. Getting Started

This landing page is built using standard HTML5, CSS3, and Vanilla JavaScript. It requires no build tools, frameworks, or package managers to run. You can edit the files directly in any code editor.

File Structure

  • index.html: The main structure and content of the page.
  • styles.css: All visual styling, including responsive design and animations.
  • script.js: Interactive functionality (mobile menu, modals, scroll behavior).
Pro Tip: Before making major changes, duplicate the folder to create a backup of the original state.

2. Global Styles & Branding

The visual identity of the site is controlled by CSS variables. Changing these values will update colors and fonts across the entire site instantly.

Color Palette

File: styles.css (Lines 1-15)

Locate the :root selector at the top of the CSS file to change the primary brand colors.

:root {
    --primary-color: #0066cc;  /* Main brand color (Buttons, Links, Highlights) */
    --secondary-color: #00b894; /* Success states, secondary accents */
    --accent-color: #f39c12;    /* Warnings, stars, highlights */
    --dark-text: #2c3e50;       /* Headings and main text */
    --light-text: #7f8c8d;      /* Body paragraphs */
    --light-bg: #f8f9fa;        /* Section backgrounds */
}

Typography

The template uses "Segoe UI" as the primary font stack. To use a custom font (like Google Fonts):

  1. Add the Google Fonts <link> tag to the <head> of index.html.
  2. Update the font-family property in the body selector in styles.css.

3. Navigation Bar

The navigation bar features a modern "Glassmorphism" effect and a responsive mobile drawer.

Logo Customization

File: index.html (Line 35)

The logo is currently an icon and text. You can replace the inner HTML of .nav-logo with an <img> tag if you prefer a graphic logo.

<!-- Current Text Logo -->
<a href="#" class="nav-logo">
    <i class="fas fa-heart-pulse"></i>
    <span>SmartCare</span>
</a>

<!-- Image Logo Replacement -->
<a href="#" class="nav-logo">
    <img src="logo.png" alt="SmartCare Logo" height="40">
</a>

Mobile Menu Configuration

File: styles.css (Media Query @ max-width: 1100px)

The mobile menu triggers at screen widths below 1100px. This is a custom breakpoint designed to accommodate the number of menu items. If you remove menu items, you can lower this breakpoint in the CSS.

Note: The mobile menu includes a footer with contact info and social icons. Ensure these are updated in index.html inside the .mobile-nav-footer div.

4. Hero Section

The Hero section is the most critical part of the page for conversion. It uses a split layout with text on the left and a visual on the right.

Copywriting Guidelines

  • Headline (H1): Keep it under 10 words. Focus on the primary benefit (e.g., "Peace of Mind").
  • Subheadline (P): Explain how you deliver the benefit in 1-2 sentences.
  • CTA Button: Use action-oriented text (e.g., "Get Started Free", "Schedule Demo").

Trust Indicators

File: index.html (Line 165)

The "Trusted by 5k+ families" section builds immediate credibility. You can update the avatar images by changing the src attributes in the .trust-avatars container.

5. Features & Benefits

This template includes two distinct sections for explaining the product:

"How It Works" (Process Steps)

Located in #how-it-works. This is a 4-step grid. Each step consists of an icon, a number, a title, and a description.

  • Icons: Uses FontAwesome classes (e.g., fas fa-box-open). Search FontAwesome Free for alternatives.
  • Responsiveness: The connecting lines between steps are automatically hidden on mobile devices via CSS.

"Benefits" (Alternating Layout)

Located in #benefits. This section uses an alternating Left-Image/Right-Text layout. To add a new benefit row:

  1. Copy an entire <div class="benefit-row"> block.
  2. Paste it after the last row.
  3. If you want the image on the right, add the class reverse to the row: <div class="benefit-row reverse">.

6. Interactive Modals

The "Schedule Demo" modal (#demoModal) has been completely redesigned with a split-screen layout (Visual Left + Form Right).

Customizing the Modal Form

File: index.html (Inside #demoModal)

The form uses standard HTML input types. To add a new field, duplicate an existing .form-group:

<div class="form-group">
    <label>New Field Label</label>
    <div class="input-wrapper">
        <i class="fas fa-tag input-icon"></i> <!-- Choose appropriate icon -->
        <input type="text" placeholder="Enter details...">
    </div>
</div>

Modal Responsiveness

The modal automatically switches to a single-column layout on mobile devices (under 768px). The visual side is reduced in height, and extra text is hidden to prioritize the form.

7. Scripts & Functionality

The script.js file handles all client-side logic.

Key Functions

  • toggleMobileMenu(): Opens/closes the side drawer navigation.
  • openModal(modalId): Opens a modal by ID (e.g., 'demo', 'contact').
  • scrollToSection(id): Smooth scrolls to a section and closes the mobile menu if open.

Analytics Integration

The scrollToSection function includes a placeholder for Google Analytics events:

if (window.gtag) {
    gtag("event", "navigation_click", {
        section: sectionId,
    });
}

Ensure you have the Google Analytics script installed in the <head> for this to work.

8. Deployment Checklist

Before publishing your landing page, ensure you have completed the following:

  • Update Title & Meta Tags: Edit the <title> and <meta name="description"> in index.html for SEO.
  • Check Links: Ensure all CTA buttons and navigation links point to the correct sections or external pages.
  • Replace Images: Replace all Unsplash placeholder images with your own assets.
  • Test Forms: Ensure your form submission logic (backend) is connected. The current forms are frontend-only templates.
  • Mobile Test: Open the site on a real mobile device to verify the navigation drawer and modal responsiveness.
